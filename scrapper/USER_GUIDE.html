1 · The big picture

The scraper turns "go find me sales leads" into a clean spreadsheet, automatically.

You tell it what to find (contractors or vendors) and where (Florida or Tennessee). It then searches Google Maps, removes duplicates and businesses you don't want (like lumber yards), adds phone/email/license details, and saves a final list you can export.

The golden rule of this upgrade Everything that worked before for Florida contractors still works exactly the same. Tennessee and Vendors were added on top — nothing old was changed or removed.

Two new things you'll notice

  • A "Vendors" section in the menu, separate from Contractors. Vendors are the material distributors you sell to (drywall/gypsum suppliers).
  • Tags on every record — small labels like state: TN Tier 1 contractor that tell you exactly what a row is and let you filter the list.

2 · Three kinds of runs

On the Dashboard you choose two things — What (Contractors / Vendors) and Where (Florida / Tennessee). That gives three valid combinations:

RunWhat it findsStatus
Florida · Contractors Drywall/construction contractors across Florida cities — the original product. OLD · unchanged
Tennessee · Contractors Contractors in Tennessee, searched around your vendor locations (new geography logic). UPDATED
Tennessee · Vendors Drywall-material distributors (GMS, L&W Supply, etc.) around Tennessee city centers. NEW
Important order for Tennessee For Tennessee, run Vendors first, then Contractors. The vendor locations become the anchor points that the contractor search circles around. (Full explanation in sections 6 & 7.)
Why is "Florida · Vendors" missing? Vendor scraping is a Tennessee-only feature by design. If you switch to Vendors mode, Florida is automatically disabled — that is expected, not a bug.

3 · The Dashboard — every setting

This is the screen where you start runs and tune settings. Here is each part, top to bottom, with what happens when you use it.

Max records (testing limit)

Dashboard › Max records to discover
A number box. It caps how many businesses the very first search step collects.

Click/type a small number (e.g. 10) → quick test run, finishes fast, low cost.
Leave it high / blank → a full production run. Use small numbers while testing, then raise it for the real run.

Data source toggles (Apify · BBB · Apollo)

Dashboard › Data sources
Three on/off switches:
• Apify — Google Maps (Discovery): the main "find businesses" engine.
• Apify — BBB (Enrichment): adds Better Business Bureau ratings/accreditation.
• Apollo (Enrichment): looks up business email addresses.

Turn one off → that step is skipped on the next run (e.g. turn off Apollo to skip email lookup and save credits). All on → the most complete data.

Tennessee search radii (miles) TN only

Dashboard › Tennessee search radii
Two number boxes. They control how wide the Tennessee searches reach.
SettingDefaultWhat it controls
Vendor radius20 milesWhen scraping vendors, how far around each Tennessee city center to look.
Contractor radius50 milesWhen scraping TN contractors, how far around each vendor location to look.

These are two different radii because they're used by two different searches. They only affect Tennessee — Florida ignores them completely. Changes apply to the next run.

Statewide TN license verify (optional) TN only

Dashboard › Statewide TN license verify
A checkbox. Tennessee has no single free statewide license file, so this is off by default.

Leave unchecked (default) → fast TN runs using the free Nashville license data.
Check it → also cross-checks the state verify site for extra license matches. Slower. Florida is unaffected either way.

Save settings button

Saves the radii / verify / source choices. The button only lights up when you've actually changed something, so you always know if there are unsaved edits.

Run scope — What & Where

Dashboard › Run scope
Two rows of buttons:
• What:  Contractors  Vendors
• Where:  Florida  Tennessee

Click "Vendors" → Where automatically jumps to Tennessee and Florida greys out (vendors are TN-only).
Click "Contractors" → you can pick either Florida or Tennessee.

Helper note you'll see When you pick Tennessee + Contractors, the dashboard shows a reminder: the run searches the contractor radius around your vendor accounts. If no vendors exist yet, it falls back to searching TN cities directly (Florida-style).

The Start button

The big button reads exactly what you're about to do, e.g. "Start TN Vendor Scrape" or "Start FL Contractor Scrape". Click it and the run begins; you can watch progress on the Pipeline Stages and Logs pages.

5 · Florida contractors OLD — unchanged

This is the original product. It behaves exactly as it always did.

When you run Florida · Contractors:

  1. The tool goes through the Florida cities list and their ZIP codes.
  2. It searches Google Maps in each of those areas for drywall/construction contractors.
  3. It removes duplicates, drops lumber businesses, adds details, and saves them to the Contractors page.
Nothing here changed Same cities, same keywords, same results as before the upgrade. Florida does not use vendors, radii, or the TN license verify. Old records may show no state tag — that's normal; they are treated as Florida.

6 · Tennessee contractors UPDATED logic

Tennessee contractors are found using a new "vendors first" geography — not the Florida city method.

The new idea: search around your vendors

In Tennessee, your best contractor leads are the ones near the distributors that sell materials. So instead of sweeping whole cities, the tool draws a circle (the contractor radius, default 50 miles) around each vendor location you already scraped, and searches inside those circles.

This is why order matters The vendor-geography logic only runs if vendors already exist. So the correct sequence is: (1) run Tennessee Vendors → (2) then run Tennessee Contractors.

What if you skip the vendor run?

If there are no vendors yet, the tool doesn't fail — it falls back to the Florida-style method: it searches the Tennessee cities and their ZIPs directly, just like Florida does. You still get contractors; they just aren't anchored to vendor locations.

How to tell which one ran The Logs page tells you. If you see a line about the Florida-style / city fallback during a TN contractor run, it means no vendors were available, so the vendor-geography logic was skipped. To get the intended behaviour, run a vendor scrape first and try again.

7 · Tennessee vendors NEW

Vendors are the material distributors you sell to — and they live in their own section.

What a "vendor" is here

A vendor = a drywall / gypsum / building-material distributor (e.g. GMS, L&W Supply). These are different from contractors, so they are stored on a separate Vendors page, not mixed into Contractors.

How the vendor run works

  1. It takes the Tennessee city centers and draws the vendor radius (default 20 miles) around each.
  2. It searches Google Maps inside those circles for material distributors.
  3. It keeps only the real distributors and drops contractors that look like suppliers (e.g. "S & L Carpentry" is a contractor, not a vendor — it's removed).
  4. It rolls related branches up under one network name (many local GMS branches → one "GMS"), and flags large chains as big-box.
  5. The clean distributor list is saved to the Vendors page.
Why vendors are separate Contractors and vendors are two different audiences. Keeping them in separate pages/tabs means a vendor never accidentally shows up in your contractor sales sheet, and the vendor list is also what powers the Tennessee contractor geography (section 6).

What the Vendors page shows

A table just like Contractors — distributor name, Network, Type (or a big-box badge), address, city, Tier, ZIP, phone, email, website — with a filter box on every column and an Export CSV button.

10 · Click a record — the detail panel

Click any contractor or vendor row and a panel slides in from the side, explaining exactly what that record is and where its data came from.

Contractors / Vendors › click a row → detail drawer

Tags section

Shows every label on this record as badges — contractor TN Nashville Tier 1 37203 GMS — so you instantly understand what the business is and how it was categorised.

Sources (raw layer) section

Lists each original source row that fed this record — e.g. the Google Maps result, the BBB listing, the license match. This is the provenance: it answers "where did this data come from?" Every source is linked to the record by its canonical_entity_id.

Classification section (contractors only)

For contractors, an extra "why was this included" audit trail. (Vendors don't show this — it's contractor-specific.)

So the flow is: see a row → click it → read its Tags (what it is) and its Sources (where it came from). That's the whole "data separation" idea made visible — one clean record on top, its raw source rows underneath.

Why some rows "disappear" from exports

The final export is the clean sales list, so by default it hides:

  • excluded_reason rows — mostly lumber businesses, which you don't sell to.
  • out_of_territory rows — businesses outside your selling area.

They aren't deleted — they're tagged and set aside. You can still include them for auditing by exporting with the "include excluded" option.

About lumber "Lumber" businesses (lumber yards, sawmills, etc.) are intentionally excluded because they aren't drywall-material targets. The tool is careful here — for example "plumber" is not treated as lumber. Removed rows always show a clear excluded_reason tag so you know why.

13 · Quick answers

"I ran a Tennessee contractor scrape and got vendors / odd results."

Check the Logs. If it used the Florida-style fallback, no vendors existed yet. Run a Tennessee Vendor scrape first, then run contractors again so the vendor-geography logic can work.

"Why is Florida disabled?"

You're in Vendors mode. Vendors are Tennessee-only. Switch to Contractors to enable Florida.

"My Florida results look the same as always — is that right?"

Yes. Florida was deliberately left untouched. All the new behaviour is Tennessee/Vendor only.

"What's the difference between Tier 1 and Tier 2?"

Tier is how important a city/market is — Tier 1 is top priority. It's just a label to help you focus; it doesn't remove anything.

"Where did this email/phone come from?"

Click the record → open the Sources section. Each source row shows which provider supplied which data.

"How do I get a clean list to hand to sales?"

Open Contractors (or Vendors), filter if you like, and click Export CSV. Lumber and out-of-territory rows are removed automatically.

This guide describes the dashboard as an end user sees it. Engineering details (data sources, internal IDs, license providers) are intentionally kept light here — the goal is to help you run scrapes and read results with confidence.
